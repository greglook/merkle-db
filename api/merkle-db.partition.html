merkle-db.partition

Partitions contain non-overlapping ranges of the records witin a table. Partition nodes contain metadata about the contained records and links to the tablets where the data for each field family is stored.

apply-patch!

(apply-patch! store part changes)

Performs an update across the tablets in the partition to merge in the given patch changes. Returns a sequence of zero or more partitions.

from-records

(from-records store parameters records)

Constructs new partitions from the given map of record data. The records will be split into tablets matching the given families, if provided. Returns a sequence of persisted partitions.

join

(join store left right)

Join two partitions into a single partition. The partition key ranges must not overlap.

read-all

(read-all store part fields)

Read a lazy sequence of key/map tuples which contain the requested field data for every record in the partition.

view source

read-batch

(read-batch store part fields record-keys)

Read a lazy sequence of key/map tuples which contain the requested field data for the records whose keys are in the given collection.

view source

read-range

(read-range store part fields start-key end-key)

Read a lazy sequence of key/map tuples which contain the field data for the records whose keys lie in the given range, inclusive. A nil boundary includes all records in that range direction.

view source

split

(split store part split-key)

Split the partition into two partitions at the given key. All records less than the split key will be contained in the first partition, all others in the second.
